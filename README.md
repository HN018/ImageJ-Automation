# ImageJ Automation Pipelines for Fluorescence Microscopy

## Overview
This repository contains a suite of ImageJ/Fiji macros designed to automate the processing, extraction, and standardization of multi-channel fluorescence microscopy data. They are separated to individual modules for customized use.

When investigating complex cellular mechanisms—such as lysosomal function, lipid metabolism, and mitophagy—manual image processing introduces human error and creates massive data bottlenecks. These scripts were developed to replace manual GUI interactions with automated, reproducible pipelines. This ensures that downstream statistical analyses and quantification (handled via Python/pandas) are based on cleanly segmented, standardized, and unbiased image data. 

## The Problem
Quantifying organelle dynamics across large datasets requires repetitive processing: splitting channels, standardizing contrast for visualization, and rigorously saving Regions of Interest (ROIs) to ensure reproducibility. Doing this manually across >3 biological replicates and 50+ cells per group is inefficient and prone to errors.

## The Solution (Macros)
These macros automate the most tedious and error-prone steps of the image analysis pipeline, ensuring robust data extraction.

**Automated ROI Saver (`save_rois.ijm`)**: Ensures strict data provenance and record keeping by automatically detecting the active image's directory, creating a standardized `ROI` parent folder, and saving the ROI Manager contents (as `.roi` or `.zip` depending on the count) as the correponding image's name without manual file routing.
**Batch Channel Splitter (`split_channels_batch.ijm`)**: Iterates through all open images in the workspace, identifies multi-channel files, and executes a batch channel split, ignoring single-channel images. 
**Batch TIFF Saver (`save_all_tiff.ijm`)**: A rapid data-dump utility that takes all open, processed workspace images, strips invalid filename characters, saves them as standardized TIFFs to a target directory, and safely closes the workspace.
**Composite Contrast Standardizer (`composite_export.ijm`)**: Automatically formats dual-channel images into a standardized composite view (assigning colored channels, example showed 2 channels) and applies a uniform 0.2% saturated contrast enhancement for consistent visual inspection of co-localization.
**Stack Duplicator (`create_stack.ijm`)**: A utility script that takes a single frame and rapidly generates a standardized stack, useful for generating movies from images of each frame (example showed 45 slices).

## Usage
1. Download the `.ijm` files and simply drag them in to your imageJ window (FIJI release preferred). Alternatively you can install them as plugins.
2. For batch processing scripts (like the Channel Splitter or TIFF Saver), ensure your target images are open in the ImageJ workspace before execution.
3. For the ROI Exporter, the source image must be saved to a local directory first, as the script programmatically builds the ROI archive relative to the parent image's file path.

## Downstream Integration
The standardized TIFFs and ROIs generated by this pipeline are designed to be fed into downstream automated intensity quantification scripts. By standardizing the input data architecture here, subsequent Python (pandas) scripts can calculate intensities, track coordinates, and perform statistical evaluations across experimental conditions seamlessly.

## Further extension
The modular nature of this ImageJ suite allows it to serve as a preprocessing backbone for advanced computer vision workflows. To further scale throughput, the classical ROI generation steps can be augmented using machine learning-based instance segmentation algorithms (e.g., Cellpose) for morphology-aware cell masking. Due to the need for specialized training for each application, such modules are not included in this repository.
